# Installation

This guide covers installing CHS Platform for local development. For production deployments using Docker, see [Docker Setup](/getting-started/docker-setup).

## Clone the Repository

```bash
git clone https://github.com/smellak/chs-platform-v2.git
cd chs-platform-v2
```

## Install Dependencies

The project uses npm workspaces. Install all dependencies from the root:

```bash
npm install
```

This installs dependencies for all workspaces:

- `apps/platform` -- The main Next.js application
- `apps/docs` -- This documentation site
- `packages/sdk` -- The integration SDK
- `packages/agent-sdk` -- The Agent SDK
- `packages/db` -- Database schema and migrations
- `packages/auth` -- Authentication utilities
- `packages/config` -- Shared configuration
- `packages/ui` -- Shared UI components

## Set Up the Database

### 1. Create a PostgreSQL Database

```bash
# Using psql
createdb aleph

# Or connect to PostgreSQL and create it
psql -U postgres
CREATE DATABASE aleph;
CREATE USER aleph WITH PASSWORD 'aleph_secret';
GRANT ALL PRIVILEGES ON DATABASE aleph TO aleph;
\q
```

### 2. Configure Environment Variables

Create a `.env` file in the `apps/platform` directory:

```bash
# apps/platform/.env
DATABASE_URL=postgresql://aleph:aleph_secret@localhost:5432/aleph
JWT_SECRET=your-secret-key-at-least-64-characters-long-for-production-use
DOMAIN=localhost
NODE_ENV=development
```

### 3. Run Migrations

```bash
npm run db:migrate
```

### 4. Seed the Database

```bash
npm run db:seed
```

This creates:
- A default organization ("Centro Hogar Sanchez")
- Eight departments (Compras, Ventas, Logistica, Marketplace, Marketing, Contenido, E-commerce, IT)
- Four system roles (Super Admin, Dept Admin, User, Viewer)
- Three sample users (admin, carlos.martinez, ana.rodriguez)
- Two sample applications with access policies
- API provider configurations (Anthropic, OpenAI, Google AI)

## Build All Packages

```bash
npm run build
```

This uses Turborepo to build all packages in the correct dependency order:

1. `@chs-platform/db` -- Database schema
2. `@chs-platform/auth` -- Auth utilities
3. `@chs-platform/sdk` -- SDK
4. `@chs-platform/agent-sdk` -- Agent SDK
5. `apps/platform` -- Main application

## Start the Development Server

```bash
npm run dev
```

The platform will be available at [http://localhost:3000](http://localhost:3000).

## Verify the Installation

1. Open [http://localhost:3000](http://localhost:3000) in your browser
2. You should see the login page
3. Log in with the default credentials: `admin` / `admin123`
4. You should see the admin dashboard
