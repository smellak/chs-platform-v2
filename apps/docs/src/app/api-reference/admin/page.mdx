# Admin Endpoints

The admin API provides CRUD operations for managing platform entities. All admin endpoints require authentication and appropriate permissions (typically `super-admin` or `dept-admin` role).

## Users

### GET /api/admin/users

List all users in the organization.

**Query Parameters:**
- `search` -- Filter by name, email, or username
- `departmentId` -- Filter by department

**Response (200):**
```json
{
  "users": [
    {
      "id": "uuid",
      "username": "admin",
      "email": "admin@centrohogar.es",
      "firstName": "Admin",
      "lastName": "Principal",
      "isActive": true,
      "isSuperAdmin": true,
      "lastLogin": "2026-03-07T10:00:00Z",
      "departments": [
        {
          "id": "uuid",
          "name": "IT",
          "role": "super-admin"
        }
      ]
    }
  ]
}
```

### POST /api/admin/users

Create a new user.

**Request:**
```json
{
  "username": "juan.garcia",
  "email": "juan@centrohogar.es",
  "firstName": "Juan",
  "lastName": "Garcia",
  "password": "secure-password",
  "departmentId": "uuid",
  "roleId": "uuid"
}
```

**Response (201):**
```json
{
  "user": {
    "id": "uuid",
    "username": "juan.garcia",
    "email": "juan@centrohogar.es"
  }
}
```

## Common Error Responses

All admin endpoints return consistent error formats:

**401 Unauthorized:**
```json
{
  "error": "Authentication required"
}
```

**403 Forbidden:**
```json
{
  "error": "Admin access required"
}
```

**404 Not Found:**
```json
{
  "error": "Resource not found"
}
```

**400 Bad Request:**
```json
{
  "error": "Validation error message"
}
```
