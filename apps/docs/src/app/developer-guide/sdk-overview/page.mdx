# SDK Overview

The `@chs-platform/sdk` package enables external applications to integrate with CHS Platform. It provides header parsing, authentication middleware, and helper functions for both Express and Next.js applications.

## Installation

```bash
npm install @chs-platform/sdk
```

## Package Entry Points

The SDK provides three entry points:

| Import | Description |
|--------|-------------|
| `@chs-platform/sdk` | Core exports: `parseCHSHeaders`, `getCHSLoginUrl`, types |
| `@chs-platform/sdk/express` | Express middleware functions |
| `@chs-platform/sdk/next` | Next.js helper functions |

## Core Exports

### `parseCHSHeaders(headers)`

Parses raw HTTP headers into an `CHSUser` object. Returns `null` if the `x-chs-user-id` header is missing.

```typescript
import { parseCHSHeaders } from "@chs-platform/sdk";

const user = parseCHSHeaders(request.headers);

if (user) {
  console.log(user.id);          // UUID
  console.log(user.name);        // "Carlos Martinez"
  console.log(user.email);       // "carlos@centrohogar.es"
  console.log(user.orgId);       // UUID
  console.log(user.orgName);     // "Centro Hogar Sanchez"
  console.log(user.dept);        // "Logistica y Almacen"
  console.log(user.deptId);      // UUID
  console.log(user.role);        // "user"
  console.log(user.accessLevel); // "full"
  console.log(user.permissions); // { "apps.read": true }
}
```

**Parameters:**
- `headers: Record<string, string | string[] | undefined>` -- A headers object (supports Express `req.headers`, Next.js headers, or plain objects)

**Returns:** `CHSUser | null`

### `getCHSLoginUrl(platformUrl, redirectUrl?)`

Generates a URL that redirects the user to the CHS Platform login page with a redirect parameter.

```typescript
import { getCHSLoginUrl } from "@chs-platform/sdk";

const loginUrl = getCHSLoginUrl("https://platform.example.com", "https://myapp.example.com/dashboard");
// Returns: "https://platform.example.com/login?redirect=https%3A%2F%2Fmyapp.example.com%2Fdashboard"
```

**Parameters:**
- `platformUrl: string` -- The base URL of CHS Platform
- `redirectUrl?: string` -- Where to redirect after login (defaults to `window.location.href` in browser environments)

**Returns:** `string`

## Types

### `CHSUser`

```typescript
interface CHSUser {
  id: string;
  name: string;
  email: string;
  orgId: string;
  orgName: string;
  dept: string;
  deptId: string;
  role: "super-admin" | "dept-admin" | "user" | "viewer";
  accessLevel: "full" | "readonly";
  permissions: Record<string, boolean>;
}
```

### `CHSHeaders`

```typescript
interface CHSHeaders {
  "x-chs-user-id": string;
  "x-chs-user-name": string;
  "x-chs-user-email": string;
  "x-chs-org-id": string;
  "x-chs-org-name": string;
  "x-chs-dept": string;
  "x-chs-dept-id": string;
  "x-chs-role": string;
  "x-chs-access-level": string;
  "x-chs-permissions": string;
}
```

## Package Structure

```
@chs-platform/sdk/
├── dist/
│   ├── index.js            # CJS entry
│   ├── index.mjs           # ESM entry
│   ├── index.d.ts          # Type declarations
│   └── middleware/
│       ├── express.js      # CJS Express middleware
│       ├── express.mjs     # ESM Express middleware
│       ├── express.d.ts    # Express type declarations
│       ├── next.js         # CJS Next.js helpers
│       ├── next.mjs        # ESM Next.js helpers
│       └── next.d.ts       # Next.js type declarations
```

The package is built with [tsup](https://tsup.egoist.dev/) and supports both CommonJS and ESM imports.
