# SDK Overview

The `@aleph-platform/sdk` package enables external applications to integrate with the Aleph Platform. It provides header parsing, authentication middleware, and helper functions for both Express and Next.js applications.

## Installation

```bash
npm install @aleph-platform/sdk
```

## Package Entry Points

The SDK provides three entry points:

| Import | Description |
|--------|-------------|
| `@aleph-platform/sdk` | Core exports: `parseAlephHeaders`, `getAlephLoginUrl`, types |
| `@aleph-platform/sdk/express` | Express middleware functions |
| `@aleph-platform/sdk/next` | Next.js helper functions |

## Core Exports

### `parseAlephHeaders(headers)`

Parses raw HTTP headers into an `AlephUser` object. Returns `null` if the `x-chs-user-id` header is missing.

```typescript
import { parseAlephHeaders } from "@aleph-platform/sdk";

const user = parseAlephHeaders(request.headers);

if (user) {
  console.log(user.id);          // UUID
  console.log(user.name);        // "Carlos Martinez"
  console.log(user.email);       // "carlos@centrohogar.es"
  console.log(user.orgId);       // UUID
  console.log(user.orgName);     // "Centro Hogar Sanchez"
  console.log(user.dept);        // "Logistica y Almacen"
  console.log(user.deptId);      // UUID
  console.log(user.role);        // "user"
  console.log(user.accessLevel); // "full"
  console.log(user.permissions); // { "apps.read": true }
}
```

**Parameters:**
- `headers: Record<string, string | string[] | undefined>` -- A headers object (supports Express `req.headers`, Next.js headers, or plain objects)

**Returns:** `AlephUser | null`

### `getAlephLoginUrl(platformUrl, redirectUrl?)`

Generates a URL that redirects the user to the Aleph Platform login page with a redirect parameter.

```typescript
import { getAlephLoginUrl } from "@aleph-platform/sdk";

const loginUrl = getAlephLoginUrl("https://platform.example.com", "https://myapp.example.com/dashboard");
// Returns: "https://platform.example.com/login?redirect=https%3A%2F%2Fmyapp.example.com%2Fdashboard"
```

**Parameters:**
- `platformUrl: string` -- The base URL of the Aleph Platform
- `redirectUrl?: string` -- Where to redirect after login (defaults to `window.location.href` in browser environments)

**Returns:** `string`

## Types

### `AlephUser`

```typescript
interface AlephUser {
  id: string;
  name: string;
  email: string;
  orgId: string;
  orgName: string;
  dept: string;
  deptId: string;
  role: "super-admin" | "dept-admin" | "user" | "viewer";
  accessLevel: "full" | "readonly";
  permissions: Record<string, boolean>;
}
```

### `AlephHeaders`

```typescript
interface AlephHeaders {
  "x-chs-user-id": string;
  "x-chs-user-name": string;
  "x-chs-user-email": string;
  "x-chs-org-id": string;
  "x-chs-org-name": string;
  "x-chs-dept": string;
  "x-chs-dept-id": string;
  "x-chs-role": string;
  "x-chs-access-level": string;
  "x-chs-permissions": string;
}
```
