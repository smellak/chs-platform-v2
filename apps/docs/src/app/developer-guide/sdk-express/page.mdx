# Express Middleware

The `@chs-platform/sdk/express` entry point provides middleware functions for Express applications. These middleware functions parse CHS headers, enforce authentication, and check permissions.

## Installation

```bash
npm install @chs-platform/sdk
```

## Middleware Functions

### `chsMiddleware()`

Parses CHS headers from the incoming request and attaches the user information to `req.chs`. This middleware does not block requests -- it simply makes user information available if present.

```typescript
import express from "express";
import { chsMiddleware } from "@chs-platform/sdk/express";

const app = express();

// Apply to all routes
app.use(chsMiddleware());

app.get("/api/data", (req, res) => {
  if (req.chs) {
    res.json({ message: `Hello, ${req.chs.name}` });
  } else {
    res.json({ message: "Hello, anonymous user" });
  }
});
```

After this middleware runs, `req.chs` will be:
- An `CHSUser` object if valid CHS headers are present
- `undefined` if the `x-chs-user-id` header is missing

### `requireCHS()`

Ensures that a valid CHS user is present on the request. Returns a 401 response if `req.chs` is not set.

```typescript
import { chsMiddleware, requireCHS } from "@chs-platform/sdk/express";

app.use(chsMiddleware());

// This route requires an authenticated CHS user
app.get("/api/protected", requireCHS(), (req, res) => {
  // req.chs is guaranteed to be defined here
  res.json({ user: req.chs });
});
```

**Response on failure:**
```json
{ "error": "CHS authentication required" }
```
Status code: `401`

### `requireCHSPermission(permission)`

Checks that the authenticated user has a specific permission. Returns 401 if no user, or 403 if the permission is not present. Super admins (`role: "super-admin"`) bypass this check.

```typescript
import {
  chsMiddleware,
  requireCHSPermission,
} from "@chs-platform/sdk/express";

app.use(chsMiddleware());

// Only users with "apps.manage" permission can access
app.post(
  "/api/admin/settings",
  requireCHSPermission("apps.manage"),
  (req, res) => {
    res.json({ message: "Settings updated" });
  }
);
```

**Response on missing user:**
```json
{ "error": "CHS authentication required" }
```
Status code: `401`

**Response on missing permission:**
```json
{ "error": "Permission 'apps.manage' required" }
```
Status code: `403`

### `requireCHSAccessLevel(level)`

Checks that the authenticated user has the required access level. Accepts `"full"` or `"readonly"`.

```typescript
import {
  chsMiddleware,
  requireCHSAccessLevel,
} from "@chs-platform/sdk/express";

app.use(chsMiddleware());

// Only users with full access can modify data
app.put(
  "/api/records/:id",
  requireCHSAccessLevel("full"),
  (req, res) => {
    res.json({ message: "Record updated" });
  }
);

// Users with readonly access can still read
app.get(
  "/api/records",
  requireCHSAccessLevel("readonly"),
  (req, res) => {
    res.json({ records: [] });
  }
);
```

**Response on failure:**
```json
{ "error": "Full access required" }
```
Status code: `403`

Note: When `level` is `"readonly"`, both `"full"` and `"readonly"` users pass the check (since `"full"` implies read access). When `level` is `"full"`, only users with `accessLevel: "full"` pass.

## Complete Example

Here is a full Express application integrated with CHS Platform:

```typescript
import express from "express";
import {
  chsMiddleware,
  requireCHS,
  requireCHSPermission,
  requireCHSAccessLevel,
} from "@chs-platform/sdk/express";

const app = express();
app.use(express.json());

// Parse CHS headers on all requests
app.use(chsMiddleware());

// Public health endpoint (no auth required)
app.get("/api/health", (req, res) => {
  res.json({ status: "healthy" });
});

// Protected route: requires any authenticated user
app.get("/api/profile", requireCHS(), (req, res) => {
  res.json({
    name: req.chs!.name,
    email: req.chs!.email,
    department: req.chs!.dept,
    role: req.chs!.role,
  });
});

// Read-only route
app.get(
  "/api/reports",
  requireCHS(),
  requireCHSAccessLevel("readonly"),
  (req, res) => {
    res.json({ reports: [] });
  }
);

// Write route: requires full access
app.post(
  "/api/reports",
  requireCHS(),
  requireCHSAccessLevel("full"),
  (req, res) => {
    res.json({ message: "Report created" });
  }
);

// Admin route: requires specific permission
app.delete(
  "/api/reports/:id",
  requireCHSPermission("apps.manage"),
  (req, res) => {
    res.json({ message: "Report deleted" });
  }
);

app.listen(5000, () => {
  console.log("App running on port 5000");
});
```

## TypeScript Types

When using TypeScript, the `req.chs` property is typed as `CHSUser | undefined`. After `requireCHS()` middleware, you can safely assert it is defined:

```typescript
app.get("/api/data", requireCHS(), (req, res) => {
  const user = req.chs!; // Safe assertion after requireCHS
  res.json({ userId: user.id });
});
```

If you want full type safety without assertions, you can use a type guard:

```typescript
app.get("/api/data", requireCHS(), (req, res) => {
  if (!req.chs) return; // TypeScript narrows the type
  res.json({ userId: req.chs.id });
});
```
